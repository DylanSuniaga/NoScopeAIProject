However, these methods are limited because they either depend on known cheats or analyze behavior at a general level rather than focusing on individual players. 
Most current anti-cheat systems rely on known patterns. Some use signature-based detection, which looks for known cheat software in memory. Others use rule-based systems, like flagging unusually high stats. More recent work looks at behavioral biometrics, where a player’s movement or input style can act like an identity signature. There are also change-point detection methods that identify sudden shifts in data, and causal inference methods that help explain whether a change is suspicious or caused by something normal. The main issue is that these approaches are usually used separately rather than as one unified system.

Key Research and Gap
Behavioral biometrics can model stable, identity-specific player behavior
Change-point detection methods like BOCPD, Page-Hinkley, and CUSUM can detect abrupt shifts
Causal inference frameworks like DoWhy can test whether changes come from confounders
Prior work shows these ideas are effective individually
Gap: existing systems rarely integrate identity modeling, shift detection, and causal validation into one pipeline 


In the research literature, behavioral biometrics has shown that user behavior can be modeled as something stable and identity-specific, not just random gameplay data. At the same time, change-point detection methods such as Bayesian Online Change-Point Detection, Page-Hinkley, and CUSUM are useful for catching sudden behavioral shifts in streaming data. Causal inference frameworks like DoWhy add another layer by checking whether those shifts might actually be explained by things like latency, hardware changes, or software updates. So the literature already shows that each of these ideas works on its own. The gap is that prior systems usually do not combine all three into one anti-cheat pipeline, and that is where our project contributes something new.

Our solution
Detect behavioral deviations, not cheat software
Learn a player-specific behavioral fingerprint from aim telemetry
Analyze causal windows over time
Detect sustained suspicious shifts with change-point detection
Apply a confounder-aware causal gate to reduce false positives
Output a moderator-friendly anomaly report

Our key idea is that every player has a unique playstyle. Instead of detecting cheats directly, we detect when a player suddenly behaves in a way that is statistically unlikely for them.

Processing Pipeline
Behavioral Fingerprint Encoder
Current implementation: lightweight MLP over causal windows to get embeddings.
Synthetic path: learns player-specific fingerprints
CSGO path: learns window-level aim structure (then logistic regression for session)
Change-Point Detection
Current implementation: Page-Hinkley
Purpose: detect the first sustained upward shift in suspiciousness
Causal Validation Layer
Current implementation: confounder-aware score discounting
Used to suppress false alarms from lag, sensitivity changes, and similar factors
The system has three main components.
First, the behavioral fingerprint encoder learns a stable representation of each player’s behavior.
Second, the change-point detection module identifies sudden shifts in behavior over time.
Third, the causal validation layer ensures that detected changes are not caused by external factors like lag or hardware changes.


Why These Features
Smooth / structured features: angular speed, acceleration, jerk, curvature, straightness, settling
Stochastic / variability features: entropy, reversals, micro-corrections, timing regularity, autocorrelation
We model behavior as smooth intentional control plus stochastic residual variation.
The encoder learns how a player normally behaves, the detector finds abnormal changes, and the causal layer checks if those changes are legitimate.
